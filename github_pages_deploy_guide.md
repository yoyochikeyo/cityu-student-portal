# GitHub Pages Deploy Guide

## What To Upload

Upload these files/folders to a GitHub repository:

- `index.html`
- `manifest.webmanifest`
- `sw.js`
- `assets/`
- `mobile_app_assignment_checklist.md`
- `mobile_app_run_guide.md`

Optional supporting files:

- `student_portal_preview.html`
- `student_portal_repackaged_project.md`

## Deploy From GitHub Website

1. Go to GitHub.
2. Create a new repository, for example `cityu-student-portal`.
3. Upload the files above into the repository root.
4. Open repository `Settings`.
5. Go to `Pages`.
6. Under `Build and deployment`, choose:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
7. Click `Save`.
8. Wait 1-3 minutes.
9. GitHub will show a public URL like:

```text
https://your-username.github.io/cityu-student-portal/
```

Open that URL on your phone.

## Phone Preview

Once GitHub Pages is active, open the GitHub Pages URL in your phone browser.

For app-like use:

- Android Chrome: menu -> Add to Home screen
- iPhone Safari: Share -> Add to Home Screen
